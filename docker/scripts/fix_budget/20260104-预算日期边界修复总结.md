# 预算日期边界修复总结

**日期**：2026-01-04  
**问题**：预算查询时未包含结束日期当天的交易  
**影响**：所有在月末最后一天有交易的用户

---

## 【问题根源】

预算的 `end_date` 字段存储的是日期的午夜零点（如 `2025-12-31 00:00:00`），但代码中使用 `<= end_date` 查询时，会排除当天的所有交易。

**示例：**
```
预算：2025-12-01 00:00:00 到 2025-12-31 00:00:00
交易：2025-12-31 00:42:00

判断：00:42:00 > 00:00:00 → 不在范围内 ❌
```

---

## 【修复方案】

### 1. 创建统一的日期工具函数

**文件**：`server/src/utils/date-helpers.ts`

```typescript
export function endOfDay(date: Date): Date {
  return new Date(
    date.getFullYear(),
    date.getMonth(),
    date.getDate(),
    23, 59, 59, 999
  );
}
```

### 2. 修复的文件

#### ✅ server/src/repositories/budget.repository.ts
- `calculateSpentAmount()` - 单人预算查询
- `calculateSpentAmount()` - 多人预算分摊查询

#### ✅ server/src/services/budget.service.ts

#### ✅ server/src/services/transaction.service.ts

- 多人预算分摊查询

## 【修复前后对比】

### 修复前
```typescript
date: {
  gte: budget.startDate,
  lte: budget.endDate,  // 2025-12-31 00:00:00
}
// 结果：排除12月31日的交易
```

### 修复后
```typescript
date: {
  gte: budget.startDate,
  lte: endOfDay(budget.endDate),  // 2025-12-31 23:59:59.999
}
// 结果：包含12月31日的所有交易
```

    u.email,
    u.name,

    COUNT(t.id) as last_day_transactions

```typescript
// 理想方案：预算应该存储日期范围，而不是时间戳
type Budget = {
  startDate: Date;  // 只存储日期部分
  endDate: Date;    // 只存储日期部分
};

// 查询时使用 DATE() 函数
WHERE DATE(transaction.date) >= DATE(budget.start_date)
  AND DATE(transaction.date) <= DATE(budget.end_date)
```

这样就完全避免了时间戳的边界问题。
